## src-tauri\src\llm

### 目录结构

```text
src-tauri/src/llm/
├─ mod.rs
├─ types.rs
└─ providers/
   ├─ mod.rs
   └─ deepseek.rs
```

### 重点说明

1. 统一入口在 `mod.rs`：`chat(provider, prompt, config)`，按 `provider` 路由到具体实现。
2. 统一返回在 `types.rs`：`LlmResponse { provider, model, content }`，上层不关心厂商差异。
3. 具体实现在 `providers/deepseek.rs`：
   - 从 `config` 读取参数（必填 `api_key`，可选 `model`、`base_url`）
   - 调用 DeepSeek `/chat/completions`
   - 解析 `choices[0].message.content`
   - 转成 `LlmResponse` 返回

### 调用示范：

```rust
        let mut config = HashMap::new();
        config.insert("api_key".to_string(), "xxxxxxx".to_string());
        config.insert("model".to_string(), "deepseek-chat".to_string());

        let result = tauri::async_runtime::block_on(chat(
            "deepseek".to_string(),
            "请回复：连接成功".to_string(),
            config,
        ));

         print!("result: {:?}", result);
```

## src-tauri\src\models\appstore.rs

### 作用

[`AppStore`](src-tauri/src/models/appstore.rs:9) 是 Tauri 应用级共享状态，统一挂在 [`manage()`](src-tauri/src/lib.rs:12) 上。

### 当前职责

1. **聊天接口**
   - 提供 [`AppStore::chat()`](src-tauri/src/models/appstore.rs:17)
   - 调用方只传 `prompt`
   - 方法内部从 [`config.llm`](src-tauri/src/models/config.rs:13) 取配置，再转调 [`llm::chat()`](src-tauri/src/llm/mod.rs:8)

2. **历史记录**
   - 由 [`chat_history`](src-tauri/src/models/appstore.rs:11) 持有
   - 这是后续聊天能力的重要基础
   - 当前先保留结构，暂不提前做过度设计

3. **配置**
   - 由 [`config`](src-tauri/src/models/appstore.rs:10) 持有
   - 当前重点是 [`llm`](src-tauri/src/models/config.rs:13) 配置
   - 配置负责提供模型调用所需参数，不在 LLM 模块内部直接耦合 Tauri state

### 设计取向

- 保持精简：只做一层必要桥接
- 保持边界清晰：[`AppStore`](src-tauri/src/models/appstore.rs:9) 负责状态组织，[`llm`](src-tauri/src/llm/mod.rs:8) 负责模型调用
- 暂不为历史记录和聊天流程做过多抽象，后续按需求演进
